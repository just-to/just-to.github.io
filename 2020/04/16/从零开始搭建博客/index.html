  • 如果版本过低，可以升级node：

  • 安装cnpm，用淘宝镜像。npm install -g cnpm --registry=https://registry.npm.taobao.org

  • 安装hexo

  • 安装git：google搜索git，downloads,或者敲命令：

Hexo设置

  • 先初始化hexo：sudo hexo init

  • 启动hexo：sudo hexo s，可以在本地4000端口查看hexo初始化的主页，嗯，这是新的一个开始。

  • 可以新建自己的博客，敲命令：sudo hexo n "博客名"，在blog/source/_posts下就能看到新建的md文件，所有创建的文件都会放在这里。

  • 写入博客内容，这里可以gedit打开，或者用vim打开文件，记得要用md的语法书写，写完md文件后保存。如果提示无法修改或保存，用chmod命令改一下文件的权限。

  • 回到终端/blog/目录下，生成文章：sudo hexo g

  • 清理，生成，启动(可执行最后一个命令)

就可以在本地查看新添加的文章。下面是部署到github，在浏览器上可以输入网址可以查看博客。

  • 回到终端，安装git插件：cnpm install --save hexo-deployer-git

  • 修改/blog/下的_config.yml文件，最后一行

遇到的问题

  • bug1：主题只有html，没有css效果。
  • 解决1：
    • 1.(可能)下载过后的主题命名不对。
    • 2.修改blog中 _config.yml 文件：将root:/ 修改为 root:/ xxx.guithub.io 。即在后面加上Github 主页的仓库名
  • bug2：不显示图片
  • 解决2：
    • 1.查资料：首先找到博客根目录下面的_config.yml文件，修改post_assrt_folder的值为true.接着 git bash 切换到博客根目录下面，安装第三方插件hexo-assrt-image：
      npm install hexo-asset-image --save
      接着hexo new “文章名” 新建一篇文章，会发现source/_posts/目录下面会生成一个和 markdown 文件同名的文件夹。再将当前这篇文章中需要插入的图片放到这个文件夹中，再到 markdown 中引用该文件夹中的图片，最后hexo g -> hexo d，就可以正常的查看这些图片了。
    • 操作之后，会创建同名的文件夹，但是好像文章中的图片没有显示，但是封面的图片加载啦。浏览器打开网址,竟然有封面图片了，开心哈哈。
    • [04-16] 针对图片无法显示的问题，在采用的上面的方法依然没有解决，所以采用另一种方式，聚合图床https://www.superbed.cn/，注册后将图片上传上去，然后在md中要用到图片时直接复制引用图床的链接。
      例如：！[描述] (复制图床的链接)
  • bug3：执行sudo hexo d:ERROR Deployer not found: git
  • 解决3：安装git插件sudo cnpm install --save hexo-deployer-git
  • bug4：首页显示文章的全部内容
  • 解决：
    • 文章太长，截断按钮文字(在需要截断的行增加此标记：(< !–more– >)excerpt_link: 阅读更多…
    • 如果嫌每篇文章手动加more标记比较麻烦，又不想在首页全文显示，可以把excerpt_all设置成true，这样首页只会显示文章归档。excerpt_all: false
    • 我是改为true，然后sudo hexo clean, sudo hexo g, sudo hexo d，重新打开网站就只看到文章归档
