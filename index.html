hexo博客更新

2020-04-15
  • 升级node：
  • 安装haxo
    sudo cnpm install -g hexo-cli

/home/twh/blog/source/_posts/hexo博客更新/2020-04-15 14-58-38屏幕截图.png

安装git：
搜索git，downloads
sudo add-apt-repository ppa:git-core/ppa
apt update
apt install git

hexo初始化
sudo hexo init
启动
sudo hexo s
新建博客
sudo hexo n “my first blog”
生成文章
sudo hexo g
在blog/source/_posts中能看到新建的md文件
退回到/blog/
清理，生成
sudo hexo clean
sudo hexo g
再启动一下sudo hexo s,就可以在本地查看

部署到github:
在github新建仓库，仓库名必须是用户名.github.io
回到终端：
安装git插件：cnpm install –save hexo-deployer-git
修改_config.yml文件，最后一行
deployment
deploy:
type: git
repo: https//github.com/just-to/just-to.github.io.git
branch: master
部署：sudo hexo d
出现问题：已初始化空的 Git 仓库于 /home/twh/blog/.deploy_git/.git/

*** 请告诉我你是谁。

运行

git config –global user.email “you@example.com“
git config –global user.name “Your Name”

来设置您账号的缺省身份标识。
如果仅在本仓库设置身份标识，则省略 –global 参数。
fatal: 无法自动探测邮件地址（得到 ‘twh@twh.(none)’）
FATAL Something’s wrong. Maybe you can find the solution here: https://hexo.io/docs/troubleshooting.html
解决：
git config –global user.email “twh1996x@gmail.com“
git config –global user.name “just-to”
完成，成功部署到github

更改主题：
hexo网站，找到合适的主题，下载他的github主题：
git clone 主题地址 themes/主题名
更改blog/下的_config.yml文件：
theme: 更改为新主题名
三连操作：
sudo hexo clean
sudo hexo g
sudo hexo s
就可以在https:\just-to.github.io看到博客啦。

问题：主题只有html，没有css效果
解决：下载过后的主题命名不对

问题：不显示图片
npm install hexo-asset-image –save
